# PS4 Alignment — FIFA World Cup 2026 Challenge 4: Smart Stadiums & Tournament Operations

This document maps every PS4 pillar to the specific feature, file, and route that implements it.

| Pillar | Feature | File(s) | Route(s) | Why |
|--------|---------|---------|----------|-----|
| **Navigation** | BFS shortest-path routing + accessible path variant | `backend/app/services/navigation.py` | `GET /api/v1/ai/navigation/path`, `GET /api/v1/ai/navigation/path/accessible`, `GET /api/v1/ai/navigation/zones` | Lets fans and volunteers find the fastest route between any two zones; accessible variant only uses ramped/accessible edges (no stairs) |
| **Crowd Management** | AI crowd density analysis with predictive 15-min forecasting | `backend/app/services/ai_service.py`, `backend/app/ai/gemini.py`, `backend/app/config/prompts.py` | `POST /api/v1/ai/crowd/analyze` | Gemini analyzes zone occupancy + trend history + weather to output current status, predicted_status_15min per zone, and recommended_action — lets organizers act before congestion happens, not after |
| **Accessibility** | Wheelchair-accessible path routing + sensory-friendly zone flag | `backend/app/services/navigation.py` (`find_accessible_path`, `SENSORY_FRIENDLY_ZONES`) | `GET /api/v1/ai/navigation/path/accessible` | `find_accessible_path()` only traverses edges marked accessible (ramp vs stairs); `SENSORY_FRIENDLY_ZONES` (VIP Lounge, Food Court East) are surfaced in zone API and AI Assistant answers |
| **Transportation** | AI transport advisor with weather-aware recommendations | `backend/app/services/ai_service.py`, `backend/app/ai/gemini.py`, `backend/app/services/weather.py` | `POST /api/v1/ai/transport/suggest`, `GET /api/v1/ai/weather` | Gemini recommends parking/transit/rideshare/walking based on gate, arrival time, and real-time weather (rain → covered transit, high heat → AC rideshare) |
| **Sustainability** | AI sustainability optimizer fed live weather context | `backend/app/services/ai_service.py`, `backend/app/ai/gemini.py`, `backend/app/services/weather.py` | `POST /api/v1/ai/sustainability/optimize` | When `optimize_sustainability` is called, it fetches current weather (temperature, precipitation) and injects it into the prompt — e.g. "high heat → expect increased cooling energy load" so recommendations account for external conditions |
| **Multilingual Assistance** | ArenaBot AI Assistant with 4-language support | `backend/app/services/ai_service.py`, `backend/app/ai/gemini.py`, `frontend/js/i18n.js`, `frontend/js/dashboard.js` | `POST /api/v1/ai/assistant/chat` (accepts `preferred_language` field) | User sends `preferred_language: "es"` and ArenaBot responds in Spanish; UI labels in dashboard toggle via i18n keys (EN, ES, FR, AR) |
| **Operational Intelligence** | Real-time SSE incident stream + predictive crowd forecasting + weather-aware ops notes | `backend/app/routes/events.py`, `backend/app/services/ai_service.py`, `backend/app/services/weather.py` | `GET /api/v1/events/incidents` (SSE), `POST /api/v1/ai/incident` | New incidents are pushed in real-time to connected dashboards via Server-Sent Events without polling; weather module generates `operational_note` (e.g. "Rain may cause fans to cluster under covered areas") consumed by ops team |
| **Real-time Decision Support** | Predictive crowd status (15-min horizon) + SSE live incident toasts + weather-driven recommendations | `backend/app/routes/events.py`, `backend/app/ai/gemini.py` (`predicted_status_15min`, `recommended_action`), `frontend/js/map.js`, `frontend/js/dashboard.js` | `POST /api/v1/ai/crowd/analyze`, `GET /api/v1/events/incidents`, `GET /api/v1/ai/weather` | Three-tier: (1) crowd analysis returns `predicted_status_15min` per zone, (2) new incidents stream to dashboard as toasts instantly, (3) weather data gives operators actionable context (rain, heat, wind) — all without a page refresh or poll loop |

## Additional Supporting Features

| Feature | File(s) | PS4 Relevance |
|---------|---------|---------------|
| Offline-first PWA (Service Worker + Manifest) | `frontend/js/sw.js`, `frontend/manifest.json`, `frontend/js/app.js` | Ensures stadium map and AI Assistant remain viewable in venue dead zones with weak connectivity — a major differentiator for large-crowd venues |
| Weather widget with operational note | `backend/app/services/weather.py`, `GET /api/v1/ai/weather` | Feeds every weather-aware prompt and surfaces a human-readable operational note for organizers (e.g. "Strong winds — secure loose signage") |
| Incident auto-push via SSE | `backend/app/routes/events.py`, `frontend/js/dashboard.js` | Eliminates manual refresh for incident list — organizers see new incidents as a toast within ~1 second |
| Sensory-friendly zone surfaces | `backend/app/services/navigation.py` line 69, `frontend` map info panel | "Quiet space near Gate B" queries answered via `SENSORY_FRIENDLY_ZONES` in navigation context, rendered in AI Assistant and zone panel |
